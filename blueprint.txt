Technical Execution of the Core Local Features

To get this in front of those 10 pilot agents fast, we don't need cloud hosting, API integrations with telecom giants, or complex databases. We build it offline-first using local architecture.
1. The Regex Parsing Engine (No-Cloud AI)

Instead of expensive AI models, the app uses standard, localized Regular Expressions (Regex) to parse incoming texts. Because telecom formats are static, a local script handles this instantly.
Plaintext

Incoming SMS: "You have received UGX 50,000 from 0772111111. New balance: UGX 1,450,000."
Parsed Data -> Type: Deposit | Amount: 50,000 | Network: MTN | Wallet_Balance: 1,450,000

2. The Predictive "Float Position Alert"

To turn this into an addictive tool that saves revenue, we add a simple mathematical calculation running locally on the SQLite database.
The app looks at the transaction frequency over the last rolling 7 days to calculate an Average Hourly Burn Rate for both MTN and Airtel float.

If the current live balance drops below the threshold required to survive the next 3 hours of peak traffic (usually 12:00 PM to 3:00 PM), the phone fires a high-priority local notification:

    🚨 Low Float Warning: Your Airtel float is at UGX 120,000. Based on your typical lunchtime traffic, you will run out of float in 45 minutes. Rebalance now to avoid turning away customers.

3. The "Money Outside" (Lending) Ledger

This replaces the chaotic scrap papers where agents write down cash borrowed by neighbors. It acts as a dual-column balance sheet:

    Borrowed From (My Liabilities): Cash or float taken from neighboring kiosks.

    Lent To (My Assets): Cash given out to neighboring kiosks or trusted customers.

The app uses an active reminder system. Every day at 6:00 PM, a banner updates: “You have UGX 350,000 outstanding on the street today. Taps to view who owes you.”

2. Technical Engineering: Bypassing the SMS Permission Wall

Google’s progressive security constraints on the READ_SMS permission (introduced in Android 10 and tightened in later versions) present a high risk of app rejection on the Google Play Store. To bypass this barrier, KioskControl uses a Notification Listener Service.

2.1 Kotlin Android Notification Listener Implementation

package ug.kioskcontrol.services

import android.service.notification.NotificationListenerService
import android.service.notification.StatusBarNotification
import android.content.Intent
import android.util.Log
import java.util.regex.Pattern

class TransactionNotificationListener : NotificationListenerService() {

    override fun onNotificationPosted(sbn: StatusBarNotification?) {
        if (sbn == null) return

        val packageName = sbn.packageName
        val extras = sbn.notification.extras
        val title = extras.getString("android.title") ?: ""
        val text = extras.getCharSequence("android.text")?.toString() ?: ""

        // Target designated mobile money carriers and system notifications
        if (packageName.contains("mtn") || packageName.contains("airtel") || 
            packageName.contains("telecom") || title.contains("Money") || 
            title.contains("MoMo")) {
            
            parseNotificationPayload(text, packageName)
        }
    }

    private fun parseNotificationPayload(text: String, origin: String) {
        try {
            // Pattern Match for MTN Cash In / Deposit Notifications
            val mtnDepositPattern = Pattern.compile(
                "You have received UGX (\\d{1,3}(,\\d{3})*) from (\\d+)\\. New balance: UGX (\\d{1,3}(,\\d{3})*)\\."
            )
            val matcher = mtnDepositPattern.matcher(text)

            if (matcher.find()) {
                val amountString = matcher.group(1).replace(",", "")
                val sender = matcher.group(3)
                val balanceString = matcher.group(4).replace(",", "")

                val amount = amountString.toDouble()
                val balance = balanceString.toDouble()

                broadcastTransaction(
                    type = "Deposit",
                    network = "MTN",
                    amount = amount,
                    counterparty = sender,
                    parsedBalance = balance,
                    rawMessage = text
                )
            }
        } catch (e: Exception) {
            Log.e("ParserError", "Failed parsing payment notification: ${e.message}")
        }
    }

    private fun broadcastTransaction(
        type: String, 
        network: String, 
        amount: Double, 
        counterparty: String, 
        parsedBalance: Double, 
        rawMessage: String
    ) {
        val intent = Intent("ug.kioskcontrol.TRANSACTION_DETECTED")
        intent.putExtra("type", type)
        intent.putExtra("network", network)
        intent.putExtra("amount", amount)
        intent.putExtra("counterparty", counterparty)
        intent.putExtra("parsedBalance", parsedBalance)
        intent.putExtra("rawMessage", rawMessage)
        sendBroadcast(intent)
    }
}
